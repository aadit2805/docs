Sift uses **Clerk** for authentication. The frontend uses `@clerk/nextjs`, the backend uses `@clerk/express`. Tokens are passed via `Authorization: Bearer` from client to server.

## Frontend (Next.js)

- `client/middleware.ts` — wraps the app in `clerkMiddleware()` and protects the `/` and `/plan` routes
- Sign-in / sign-up routes — `/sign-in` and `/sign-up` (configured via `NEXT_PUBLIC_CLERK_SIGN_IN_URL` / `NEXT_PUBLIC_CLERK_SIGN_UP_URL`)
- Token retrieval — `client/src/lib/auth-token.ts` calls Clerk's session API before each fetch and the result is attached as a Bearer header by `fetcher()`

```ts
// client/src/lib/api.ts
const token = await getAuthToken();
const res = await fetch(url, {
  headers: { Authorization: `Bearer ${token}` },
});
```

## Backend (Express)

- `clerkMiddleware()` runs on **all** requests so any handler can read auth state
- `requireAuth` (`server/src/middleware/auth.ts`) extracts `userId` and rejects unauthenticated requests
- Apply it only to routes that actually need a user — public endpoints (courses, professors, recommendations without history) skip it

```ts
router.get('/profile', requireAuth, async (req, res) => {
  const { userId } = req.auth;
  // ...
});
```

## Which endpoints require auth?

See the [API Reference](/api-reference/introduction) — every endpoint marks whether auth is required.

In short: anything that reads or writes user-scoped data (`/profile`, `/transcript/parse`, `/planner/*`) requires auth. Public catalog data (`/courses`, `/professors`, `/recommendations`, `/degree-plan/*`) does not.

## Rate limits

| Bucket | Window | Limit |
| --- | --- | --- |
| General API | 15 min | 300 requests |
| Transcript upload | 15 min | 10 requests |

Transcript upload is rate-limited tighter because it triggers expensive Claude API calls (PDF parsing \+ structured extraction).

<Note>
  The backend trusts the proxy (`app.set('trust proxy', 1)`) so rate limits work correctly behind Railway's load balancer. Don't remove this — without it everyone shares the same rate-limit bucket via a single proxy IP.
</Note>
