The request logging middleware writes a single JSON line to `stdout` for every HTTP request when the response finishes. Each line is a complete JSON object, making the output easy to ingest into log aggregators.

## Log format

Every request emits one JSON object with the following fields:

| Field        | Type   | Description                                                    |
| ------------ | ------ | -------------------------------------------------------------- |
| `method`     | string | HTTP method of the request (for example, `GET`, `POST`).       |
| `path`       | string | Request path, without the query string.                        |
| `status`     | number | HTTP status code sent in the response.                         |
| `durationMs` | number | Time between receiving the request and finishing the response, in milliseconds. |

### Example

```json
{"method":"GET","path":"/users/42","status":200,"durationMs":17}
```

## Disabling request logs

Set the `LOG_LEVEL` environment variable to `silent` to bypass the middleware. No log lines are written while this is set.

```bash
LOG_LEVEL=silent
```

Unset the variable or set it to any other value to re-enable logging.
