The Sift frontend is a Next.js 16 App Router app. A few conventions are worth knowing before you make changes.

## Data fetching: `fetcher` → TanStack Query

Every API call goes through one stack:

```ts
// client/src/lib/api.ts
import { API_BASE, fetcher } from "@/lib/api";

export async function getCourses() {
  return fetcher<Course[]>(`${API_BASE}/api/courses`);
}
```

```ts
// client/src/lib/queries.ts
export function useCourses() {
  return useQuery({
    queryKey: queryKeys.courses(),
    queryFn: getCourses,
  });
}
```

Rules of the road:

- **Don't bypass `fetcher()`** — it adds the Clerk auth token. Direct `fetch` calls will look anonymous to the server.
- **Don't duplicate `API_BASE`** — import it from `api.ts`.
- **Use the query-key factory** — every queryKey lives in `queries.ts`. This keeps invalidation correct.

## Tailwind v4

There is **no** `tailwind.config.ts`. Theme lives in `client/src/app/globals.css` using `@theme inline`:

```css
@theme inline {
  --color-sift-amber: oklch(0.55 0.15 30);   /* maroon primary */
  --color-sift-green: oklch(0.7 0.15 145);
  --color-sift-red:   oklch(0.6 0.2 25);
  --color-sift-blue:  oklch(0.65 0.15 230);
  --color-sift-purple:oklch(0.55 0.2 290);
  --color-sift-surface:        oklch(0.98 0 0);
  --color-sift-surface-raised: oklch(1 0 0);
}
```

Use the `sift-*` tokens directly — `bg-sift-surface`, `text-sift-amber`, etc.

## Typography

- `font-mono` is **only** for data (course codes, GPAs, percentages, credit counts)
- Labels, headings, and body copy use the default sans-serif
- A&M collegiate light theme — maroon / black / white, light background, white cards, subtle shadows

## Components

- All base components are shadcn/ui — `client/src/components/ui/`
- Add new ones with `npx shadcn@latest add <component>`
- Prefer **server components**; opt into client components only for interactivity
- Every route has a `loading.tsx` Suspense boundary with a skeleton matching the layout
- Route transitions show a `NavProgress` bar (`client/src/components/nav-progress.tsx`)

## Error messages

User-facing error messages must be **friendly and generic**. Don't leak:

- Raw API errors
- Port numbers
- "Make sure the backend is running" or other dev-facing instructions

Wrap your `error` strings before showing them.

## Completed-courses storage

Completed courses live in `localStorage` under the key `sift_completed_courses` for unauthenticated users. After sign-in, `POST /api/user/profile/migrate` ports them into the user's profile.
