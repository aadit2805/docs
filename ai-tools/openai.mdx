OpenAI's models power a wide range of assistant, coding, and multimodal workflows. This page summarizes the current lineup and helps you pick the right model for your use case.

## Model family

<CardGroup cols={2}>
  <Card title="GPT-4o" icon="brain">
    Flagship multimodal model. Best for complex reasoning, vision, and voice tasks that need the highest quality.
  </Card>
  <Card title="GPT-4o mini" icon="bolt">
    A faster, cheaper sibling of GPT-4o. A strong default for most production workloads.
  </Card>
  <Card title="o1 / o3 reasoning" icon="lightbulb">
    Reasoning-focused models for math, science, and hard coding problems that benefit from longer thinking.
  </Card>
  <Card title="GPT Image / TTS / Whisper" icon="wand-magic-sparkles">
    Specialist models for image generation, text-to-speech, and speech-to-text.
  </Card>
</CardGroup>

## Choosing a model

- **Reach for GPT-4o** when you need top-tier quality across text, vision, and audio.
- **Reach for GPT-4o mini** for day-to-day production work where latency and cost matter alongside quality.
- **Reach for an o-series reasoning model** for problems that need step-by-step deliberation.
- **Reach for the specialist models** when the task is inherently image, speech, or transcription.

## Using OpenAI with your docs

You can wire OpenAI models into your documentation workflow:

- Enable the **ChatGPT** option in your contextual menu so readers can open any page directly in ChatGPT for follow-up questions.
- Use the OpenAI API to generate summaries, examples, or FAQs from your existing content.
