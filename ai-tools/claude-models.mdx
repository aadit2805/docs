Anthropic's Claude models power a range of assistant and coding workflows. This page summarizes the current lineup and helps you pick the right model for your use case.

## Model family

<CardGroup cols={2}>
  <Card title="Claude Opus 4" icon="brain">
    Anthropic's most capable model. Best for complex reasoning, long-horizon agentic tasks, and hard coding problems.
  </Card>
  <Card title="Claude Sonnet 4" icon="bolt">
    A balanced model that trades a little capability for speed and cost. A strong default for most production workloads.
  </Card>
  <Card title="Claude Haiku" icon="feather">
    The fastest, lowest-cost tier. Ideal for high-volume classification, extraction, and lightweight chat.
  </Card>
  <Card title="Claude Code" icon="code">
    A CLI and IDE-integrated coding agent built on the Claude models. See [Claude Code setup](/ai-tools/claude-code).
  </Card>
</CardGroup>

## Choosing a model

- **Reach for Opus** when you need the highest reasoning quality — architectural planning, deep code refactors, research tasks.
- **Reach for Sonnet** for day-to-day production work where latency and cost matter alongside quality.
- **Reach for Haiku** for real-time UX, batch processing, or when the task is well-scoped and repetitive.

## Using Claude with your docs

You can wire any Claude model into your documentation workflow:

- Use the [Claude Code](/ai-tools/claude-code) setup to edit and maintain docs from your terminal.
- Enable the **Claude** option in your contextual menu so readers can open any page directly in Claude for follow-up questions.
